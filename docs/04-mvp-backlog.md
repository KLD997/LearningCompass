# MVP Backlog (Prioritized)

## P0 - Launch Critical

1. Authentication (parent + student roles)
2. Student profile creation and management
3. Subject model and per-subject level placement
4. Lesson library CRUD (hosted + external link)
5. Assignment planner (daily + weekly)
6. Student Today dashboard + continue flow
7. Lesson completion tracking
8. Basic quiz engine (MCQ + short answer)
9. Content source rights metadata and validation
10. Parent progress dashboard (completion + mastery snapshot)

## P1 - Strong MVP

11. Parent notes + observational mastery entries
12. Prerequisite and unlock rules (simple)
13. Report export (PDF summary)
14. Achievement badges/streaks (lightweight)
15. Worldview tags and filtering controls
16. Offline activity lessons + parent confirmation

## P2 - Post-MVP

17. Bias-aware history overlay prompts
18. Science method overlay templates
19. Theology/comparative religion track templates
20. Project/rubric grading for CS work
21. Transcript-style report for high school levels

## Suggested Sprint Order

- Sprint 1: Auth, students, subjects
- Sprint 2: lessons + sources + rights metadata
- Sprint 3: assignments + student today flow
- Sprint 4: assessments + mastery tracking
- Sprint 5: reports + stabilization
