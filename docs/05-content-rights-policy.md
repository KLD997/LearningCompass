# Content Rights & Usage Policy

## Purpose
This policy defines how learning content may be stored, linked, embedded, modified, and displayed in the platform.

## Classification Requirement
Every content source and lesson must include:
- ownership_type
- license_type
- source_name
- source_url
- hosting_mode
- attribution_required
- embed_allowed
- commercial_use_allowed
- modification_allowed

## Content Modes

1. **Original Hosted Content**
   - Created by platform operator or parent.
   - May be fully stored and served in-platform.

2. **Open License / Permissioned Content**
   - Allowed only when terms explicitly permit intended use.
   - License proof or permission record must be stored.

3. **External Linked Content**
   - Store metadata only.
   - Launch user to provider URL.
   - Do not mirror provider assets unless permission exists.

4. **Embedded External Content**
   - Allowed only when provider terms and technical controls permit embedding.
   - If terms conflict, revert to outbound link mode.

## Prohibited Behavior
- Scraping third-party curriculum sites for bulk republishing.
- Rehosting third-party worksheets/videos without rights.
- Presenting third-party content as platform-owned.

## Attribution
When attribution_required = true:
- Show source name and URL in lesson footer.
- Include attribution in exports where relevant.

## Compliance Workflow
1. Add source record with rights flags.
2. Verify terms (manual check).
3. Mark allowed hosting_mode.
4. Attach source to lesson.
5. Block publishing when flags are incomplete.

## Takedown Procedure
- Provide contact channel for rights claims.
- On claim receipt: temporarily disable contested lesson.
- Review documentation and respond.
- Remove or restore based on evidence.
